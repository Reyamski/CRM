# Phase 01 Context

## Intent
Build something fast and testable, not yet production-complete.

## Constraints
- Frontend only
- Fake/demo data only
- No real authentication or persistence
- Show privacy-aware behavior in the UI

## UX Priorities
- Professional broker-facing feel
- Very easy to demo
- Fast role switching
- Clear client list/detail flow
- Masked SIN by default

## Security Positioning
- Represent secure behavior in the UI, but do not fake production security claims.
- Clearly imply that real auth, encryption, and backend audit enforcement are future phases.
